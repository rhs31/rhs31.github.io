Transplant

Transplant


About the video: The white spheres that appear are points where the Magic Leap is hand tracking, as the trees "activate" upon being touched. The initial setup where I use a controller to put down the trees would not have been seen by visitors of the exhibit, so they did not need to hold the controller. When they reached the end of the exhibit, they would give two thumbs-up, which the Magic Leap One would recognize, and it would reset the exhibit for the next person. I demonstrate this at the end.

Completed: December 2018
Project Duration: ~One Month
Role(s): Developer, Programmer, Co-Designer
Team Size: Two
Technologies/Languages I Used: Magic Leap One, Unity, C#
Code/Files: Available on request
Description:
‘Transplant’ explores the concept of putting down roots in foreign environments, via #AugmentedReality and #volumetric #photogrammetry. It is our first project on the #MagicLeapOne, and premiered at Flimgate Interactive Media Festival and the Arcadia Earth exhibition, during Miami’s Art Week, December 2018.

Banyan trees, one of the natural icons of Miami, are actually native to India, brought here in the early 1900's. Likewise, more than 50% of the residents of Miami are foreign-born "transplants." And all of us have also been putting down roots in the digital realm. Transplant explores these concepts via audio interviews, volumetric photogrammetry and augmented reality.

Transplant is an interactive art installation that was shown at the Filmgate Interactive Media Festival as well as the Arcadia Earth Show, both of which took place in Miami, Florida.

I had met Felipe Aguilar of Bogota Digital Cinema at a Magic Leap interest meetup. He called me later and pitched his concept to me, and shortly afterwards, we started working on the project. Felipe provided me with Magic Leap hardware, creative direction, photogrammetrically captured 3D models, and audio, while I concurrently developed and programmed the experience with Unity and C#, while also providing my own creative input.

This project introduced me to development on the Magic Leap, and Felipe also introduced me to the concept of photogrammetry, which I was later able to do on my own to create the centerpiece for this site.

Transplant was tried by a couple of Magic Leap employees, who enjoyed it and gave positive feedback, even promoting it on their own social media. The CEO even retweeted one of their promotions.

Sugar Crash

Sugar Crash

Completed: September 2018
Project Duration: Less than a day
Role(s): Lead Developer/Programmer, Co-Designer
Team Size: Six
Technologies/Languages I Used: Unity, C#
Code/Files:
Description:
This project was the second place winner at the 2018 Magic Game Jam at Miami-Dade College. As per the rules of the game jam, we had around 20 hours to come up with a concept and build it.

I arrived at the game jam alone, and met a nice group of students who were relative newbies, looking to learn about game development. We formed a team consisting of me, two art majors (Amanda Linares and Ashley Rosero-Barros), and three game design majors (Daniel Gomez, Gabriel Marcoccia, and Maximiliano Moscato), all freshmen except for myself. At the game jam, all participants were told to write an adjective and a noun, each on separate sticky notes, and hand them to the organizers. A random adjective and a random noun were chosen and combined to form the theme to be used for all games in the jam. The resulting theme was "sticky railway." We then had about twenty hours to create an original game with this theme.

We bounced around ideas for game mechanics and art themes. One member proposed a game that would look something like Subway Surfers. I felt that an infinite-runner type game was easy enough to implement in a short period of time. I then came up with the idea of controlling a train in a candy world, which would fit the theme, as well as present an opportunity for our artists to create interesting assets.

The artists on my team were not sure what program to use for the game, so I suggested that they try MagicaVoxel, as I felt that, from my experience with the program, this was a good way to quickly create 3D game assets, while also allowing a high level of detail if desired. I'm glad I suggested this, as they were able to create some truly beautiful assets for the game despite using the program for the first time. Practically all assets used were original. Gabriel and Maximiliano were tasked with creating a main menu for the game, which they successfully did despite having little experience with Unity. Meanwhile, I worked on building the core gameplay mechanics and level design, using C# scripting and the Unity editor.

Despite many road blocks and design changes during the creation of the game, it was functional by the end of the game jam. Not perfect, but functional.

Description:
I've grouped all of these projects together because I built them by coding along with the videos in the Complete C# Unity Developer 3D course on Udemy. I do not claim ownership of the code I wrote for Project Boost, Argon Assault, or Realm Rush. However, I have included them in my portfolio because I gained valuable Unity/C# experience and knowledge by coding along with the tutorials, making sure to understand everything I was writing, and debugging things that didn't work. The linked videos are footage of my own versions of the course projects. I was able to apply this experience to projects in which I did write original code, such as Transplant and Sugar Crash.

Project Boost is a simple physics-based game in which the player tries to move a rocket to an objective while avoiding obstacles. This project taught me about version control, Unity physics, particle systems, and loading new levels, among other things.

Argon Assault is an on-the-rails starship shoot 'em up where enemies travel on their own paths, shooting the player. This project taught me about molding terrain, using the Unity timeline system, object hierarchies, and much more.

Realm Rush is a simple tower defense game, where the player must protect a base from attack by robots. The robots will spawn and use a breadth-search-first algorithm to find a path to the base. Object pooling was also taught in this module. Built into this project was a mini-course on using MagicaVoxel, a voxel editor which provides a quick and easy way to make 3D assets.

This was a great course, and I look forward to doing more Udemy courses in the future.

Tetrostairs

Tetrostairs

Completed: May 2017
Project Duration: Built over the course of a semester
Role(s): Developer, Programmer, Designer
Team Size: One
Technologies/Languages I Used: Unity, C#, Piskel
Code/Files:

Description:
This was my first Unity project, built for my game programming class in Spring of 2017. The assignment was to create a 2D game with design/gameplay of your choosing.

The idea combines the objectives of Tetris and a platformer. The player must move falling Tetris blocks to create a path to a door at a randomized point on the other side of the screen, while avoiding running into randomly located buzzsaws, or falling off of the screen. The player intentionally has a very short jump height, so it is possible to get stuck if the wrong staircase is built, in which case the level must be restarted. The goal of the game is to try to get as many wins in a row as you can without dying.

I did all of the art and animation in a sprite-editing program called Piskel.

I believe the core concept of this game is quite good, and I would someday like to rebuild it as a polished mobile game.

Portfolio Site


Completed: January 2019
Project Duration: ~Two Weeks
Role(s): Developer, Programmer, Designer
Team Size: One
Technologies/Languages I Used: HTML/CSS, Bootstrap, JavaScript, three.js, 3DF Zephyr, Blender
Code/Files:
Description:
This website was made without the use of a premade site-builder. I had previously learned about web development from The Web Developer Bootcamp, and I wanted to put those skills to the test.

I came up with the design for the site on my own. For a long time, I had an idea to put my head as a 3D model in the center of my portfolio site (I found it a bit strange that I had never seen it done), but I did not know how to do this until I was introduced to the concept of photogrammetry while working on Transplant. After lots of experimentation, I ended up using a different program than the one used in Transplant, the new one called Zephyr3D. I had my dad take over one-hundred pictures of my head in a circular pattern, and then I plugged the photos into the program, which produced a 3D .obj model which I could then edit in both Zephyr3D and in Blender. You can see the process here.

I had a bit of previous experience with three.js, a JavaScript graphics library. After a few generic tutorials on three.js, and a lot of fiddling and searching on StackOverflow, I was able to have my imported head following my cursor in the browser. I then built my portfolio site around it using HTML and CSS along with the Bootstrap framework.

MapQuest Clone

MapQuest Clone

Note: Unnamed roads are represented with a question mark, and are valid pieces of data.
Completed: December 2016
Project Duration: ~One Month (Broken into four week-long assignments)
Role(s): Programmer
Team Size: One
Technologies/Languages I Used: C++
Code/Files: As this is an academic assignment, I have made the repo private to avoid plagiarism. Available on request.
Description:
This project is a console application that takes an input of two cities in the US, then uses a variation of Dijkstra's algorithm on files containing all major cities and roads in the US, to calculate the shortest path between the two inputted cities. It then draws the route on a map of the US, which is appropriately zoomed depending on the distance between the two cities. It does this by taking data from binary files that contain information about what a map of the US would look like, and uses a graphics library with that data to draw the map and route.

This was the final project for my Algorithms class, ECE 318. ECE 118, 218, and 318 were all project-based classes, with assignments being done in C++. This is only one of many projects that I did for those classes, and more projects from those classes can be sent on request. I figured that this was the most interesting and complex project I had done for said classes.

The project was broken into four assignments, each a separate piece of the final product. You can view the assignments here (assignments 2, 3, 4, and 6).
